# ✅ **Revised Technical Design Specification (Clean Version — No Citations)**

# **Home Lab Network Design Specification — *abhaile***

## 1. Overview

The **abhaile** home lab is an IPv4-only environment composed of two Debian-based hosts using `systemd-networkd` and `systemd-resolved`, a TP-Link Omada SDN-managed switching fabric, and a fully segmented VLAN architecture. Services run in Podman using two execution patterns:

* **Rootful ipvlan-l2 containers** with individual `/32` addresses on VLAN 20
* **Rootless containers** for auxiliary functions that do not require LAN bindings

The purpose of this document is to provide a final, stable reference for:

* Physical and logical topology
* Addressing and VLAN scheme
* Security and ACL policy
* Core services and infrastructure
* Operational procedures and GitOps management

## 2.1 Compute Hosts

| Host       | Role           | Hardware                                | OS     | Notes                                 |
| ---------- | -------------- | --------------------------------------- | ------ | ------------------------------------- |
| **phobos** | Infrastructure | Intel i7-3632QM, iGPU HD4000, Coral TPU | Debian | Runs core and DMZ services            |
| **deimos** | Media          | Intel i7-3632QM, iGPU HD4000            | Debian | Media, download, transcoding services |

## 2.2 Network Devices

| ------------------ | ------------------------- | ---------------------------------- |

| Access Points      | 4× TP-Link EAP653         | Wi-Fi 6                            |

## 2.3 Port Mapping Summary

(Condensed, deduplicated for clarity)

* Gateway uplinks to the Core Switch on a trunk (native VLAN 99).
* Core Switch uplinks to access switches on trunk profiles.
* Endpoints reside on dedicated access profiles (General, IoT, Camera, etc.).
* All access ports use BPDU Guard; trunks do not.
* PoE provided for APs, cameras, and IoT devices.

# **3. Logical Network Design**

## 3.1 IPv6 Policy

IPv6 is disabled across the full environment (gateway, hosts, VLAN interfaces, Podman networks). The gateway explicitly blocks IPv6 to avoid RA/DHCPv6 leakage and unintended egress paths.

## 3.2 Addressing & VLAN Scheme

All VLANs use the scheme: `172.20.<VLAN>.0/24`, except VLAN 1 (`192.168.0.0/24`).

| VLAN | Purpose    | Subnet          | DHCP | Notes                      |
| ---- | ---------- | --------------- | ---- | -------------------------- |
| 1    | Adoption   | 192.168.0.0/24  | Yes  | Emergency use only         |
| 99   | Management | 172.20.99.0/24  | Yes  | Switches/APs/admin access  |
| 20   | Services   | 172.20.20.0/24  | Yes  | Hosts + service `/32`      |
| 30   | General    | 172.20.30.0/24  | Yes  | User devices               |
| 40   | Gaming     | 172.20.40.0/24  | Yes  | QoS priority               |
| 50   | IoT        | 172.20.50.0/24  | Yes  | Restricted; Clean DNS only |
| 60   | Cameras    | 172.20.60.0/24  | Yes  | Isolated; no WAN           |
| 70   | Cast       | 172.20.70.0/24  | Yes  | mDNS/cast bridging         |
| 80   | Guest      | 172.20.80.0/24  | Yes  | WAN + DMZ only             |
| 90   | VPN        | 172.20.90.0/24  | No   | WireGuard peers            |
| 100  | DMZ        | 172.20.100.0/24 | Yes  | Public ingress             |

### Address Allocation Rules

* `.1` gateway
* `.2–.9` infrastructure
* `.10–.29` static
* `.30–.199` DHCP/reservation
* `.200–.254` container `/32` addresses (Services VLAN only)

## 3.3 SSIDs

| SSID    | VLAN | Visibility | Notes                       |
| ------- | ---- | ---------- | --------------------------- |
| General | 30   | Visible    | Primary client SSID         |
| Gaming  | 40   | Hidden     | 5 GHz only; low-latency     |
| IoT     | 50   | Hidden     | 2.4 GHz; PMF optional       |
| Cast    | 70   | Hidden     | Chromecast/AirPlay bridging |
| Guest   | 80   | Visible    | Isolated; DMZ-only access   |
| Mgmt    | 99   | Hidden     | Admin-only                  |

## 3.4 ACL Policy

### Default

**Deny all inter-VLAN traffic**.

### Allowed Flows

* Services → Cameras for NVR
* Services ↔ IoT for MQTT
* Cast → Services for streaming targets
* Mgmt → all LAN for admin functions
* VPN → Services
* Guest → DMZ (HTTPS only)

### Denied Flows

* IoT to WAN (except DNS/NTP to internal servers)
* Cameras to WAN
* Any VLAN → Mgmt (except admin flows)
* All external DNS/DoT/DoH
* All WAN-bound NTP (except admin/VPN)
